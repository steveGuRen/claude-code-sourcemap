# Claude Code 聊天记录保存实现分析

## 1. 模块划分

Claude Code 的聊天记录保存功能主要由以下模块组成：

| 模块 | 职责 | 文件位置 |
|------|------|----------|
| 历史记录核心 | 聊天记录的存储、读取和管理 | `src/history.ts` |

## 3. 数据结构

### 3.1 核心数据类型

#### HistoryEntry

```typescript
interface HistoryEntry {
  display: string;           // 显示的文本内容
  pastedContents: Record<number, PastedContent>;  // 粘贴的内容
}
```

#### PastedContent

```typescript
type PastedContent = {
  id: number;               // 顺序数字ID
  type: 'text' | 'image';   // 内容类型
  content: string;          // 内容
  mediaType?: string;       // 媒体类型，如 'image/png'
  filename?: string;        // 图片的显示名称
  dimensions?: ImageDimensions;  // 图片尺寸信息
  sourcePath?: string;      // 图片的原始文件路径
};
```

#### LogEntry

```typescript
type LogEntry = {
  display: string;                   // 显示的文本内容
  pastedContents: Record<number, StoredPastedContent>;  // 存储的粘贴内容
  timestamp: number;                 // 时间戳
  project: string;                   // 项目路径
  sessionId?: string;                // 会话ID
};
```

#### StoredPastedContent

```typescript
type StoredPastedContent = {
  id: number;               // 顺序数字ID
  type: 'text' | 'image';   // 内容类型
  content?: string;         // 内联内容（小粘贴）
  contentHash?: string;     // 哈希引用（大粘贴）
  mediaType?: string;       // 媒体类型
  filename?: string;        // 文件名
};
```

## 4. JSONL 字段说明

聊天记录以 JSONL (JSON Lines) 格式存储在 `~/.claude/history.jsonl` 文件中，每行是一个 `LogEntry` 对象的 JSON 表示。

### 4.1 字段详解

| 字段名 | 类型 | 描述 | 必要性 |
|--------|------|------|--------|
| `display` | string | 聊天记录的显示文本 | 必需 |
| `pastedContents` | Record<number, StoredPastedContent> | 粘贴的内容，键为ID，值为内容对象 | 必需 |
| `timestamp` | number | 记录创建的时间戳（毫秒） | 必需 |
| `project` | string | 项目的根路径 | 必需 |
| `sessionId` | string | 会话ID | 可选 |

### 4.2 `pastedContents` 内部字段

| 字段名 | 类型 | 描述 | 必要性 |
|--------|------|------|--------|
| `id` | number | 粘贴内容的唯一ID | 必需 |
| `type` | 'text'  'image' | 内容类型 | 必需 |
| `content` | string | 小文本内容的内联存储 | 可选（与contentHash二选一） |
| `contentHash` | string | 大文本内容的哈希引用 | 可选（与content二选一） |
| `mediaType` | string | 媒体类型，如 'image/png' | 可选 |
| `filename` | string | 文件名 | 可选 |

## 5. 实现流程

### 5.1 记录添加流程

1. **调用 `addToHistory`**：接收一个 `HistoryEntry` 或字符串
2. **检查环境变量**：如果 `CLAUDE_CODE_SKIP_PROMPT_HISTORY` 为真，则跳过
3. **注册清理函数**：首次调用时注册，确保进程退出时将待处理记录写入磁盘
4. **调用 `addToPromptHistory`**：
   - 转换输入为 `HistoryEntry`
   - 处理 `pastedContents`：
     - 过滤掉图片内容（单独存储）
     - 小文本直接内联存储
     - 大文本计算哈希并存储引用
   - 创建 `LogEntry` 对象
   - 添加到 `pendingEntries` 数组
   - 触发异步写入

### 5.2 存储流程

1. **`flushPromptHistory`**：
   - 检查是否有待处理记录和是否正在写入
   - 调用 `immediateFlushHistory`

2. **`immediateFlushHistory`**：
   - 确保历史文件存在
   - 获取文件锁
   - 将待处理记录转换为JSON行
   - 追加到 `history.jsonl` 文件
   - 释放锁

### 5.3 读取流程

1. **`getHistory`**：
   - 获取当前项目路径和会话ID
   - 调用 `makeLogEntryReader` 读取记录
   - 优先返回当前会话的记录
   - 然后返回其他会话的记录

2. **`makeLogEntryReader`**：
   - 首先读取内存中的 `pendingEntries`
   - 然后从 `history.jsonl` 文件中倒序读取
   - 过滤掉已跳过的时间戳
   - 解析每一行并 yield 结果

## 6. 性能优化

1. **内存缓冲**：使用 `pendingEntries` 数组缓冲记录，减少磁盘I/O次数
2. **异步写入**：后台异步写入，不阻塞主线程
3. **大文本处理**：大文本通过哈希存储到 paste store，减少历史文件大小
4. **文件锁**：使用文件锁确保多进程安全
5. **批量写入**：将多个记录一次性写入，提高效率

## 7. 安全考虑

1. **文件权限**：历史文件使用 `0o600` 权限，确保只有所有者可读写
2. **并发安全**：使用文件锁避免并发写入冲突
3. **错误处理**：写入失败时记录日志但不影响主流程

```typescript
  console.log(entry.display);
  // 处理粘贴内容
  for (const [id, content] of Object.entries(entry.pastedContents)) {
    if (content.type === 'text') {
      console.log(`Pasted text #${id}: ${content.content}`);
    }
  }
}
```

## 9. 总结

Claude Code 的聊天记录保存实现采用了以下策略：

1. **JSONL 格式**：使用 JSON Lines 格式存储，便于逐行读取和写入
2. **分层存储**：
   - 小文本直接内联存储
   - 大文本通过哈希引用存储
   - 图片单独存储
3. **异步处理**：后台异步写入，提高用户体验
4. **安全可靠**：使用文件锁确保并发安全，权限控制保护隐私
5. **性能优化**：内存缓冲、批量写入等措施减少I/O开销

这种设计既保证了聊天记录的完整保存，又兼顾了性能和安全性，为用户提供了良好的历史记录体验。
